# C4 model (draft)

## Level 1 — Context

```mermaid
C4Context
title eHub System Context
Person(renter, "Renter", "Finds and books assets")
Person(host, "Host", "Lists assets")
Person(admin, "Admin", "Moderates listings")
System(ehub, "eHub", "Marketplace API + future web/mobile")
System_Ext(email, "Email provider")
System_Ext(pay, "Payment provider", "Planned")
Rel(renter, ehub, "Uses")
Rel(host, ehub, "Uses")
Rel(admin, ehub, "Uses")
Rel(ehub, email, "Sends mail")
Rel(ehub, pay, "Charges", "Planned")
```

## Level 2 — Containers

```mermaid
C4Container
title eHub Containers
Person(user, "User")
System_Boundary(ehub, "eHub") {
  Container(api, "eHub.Api", "ASP.NET Core", "HTTP, JWT, ProblemDetails")
  Container(app, "Application + Domain", ".NET", "CQRS, aggregates")
  ContainerDb(pg, "PostgreSQL", "DB", "Planned EF persistence")
  ContainerDb(redis, "Redis", "Cache", "Planned")
}
Rel(user, api, "HTTPS/JSON")
Rel(api, app, "MediatR")
Rel(app, pg, "EF Core", "Planned")
Rel(app, redis, "Cache", "Planned")
```

## Level 3 — Components (API / Application)

- **Identity** — auth, sessions, login history  
- **Catalog** — dictionaries  
- **Assets** — universal listing aggregate  
- **Booking / Payment / Notification** — planned separate modules  

See [architecture.md](architecture.md) for dependency rules.
